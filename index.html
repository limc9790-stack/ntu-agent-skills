What's the difference?

Repeat Every Chat

You have to re-type the same instructions in every new conversation. No memory between sessions.

Generic Responses

Claude gives general answers — it doesn't know your specific workflow, brand guidelines, or project conventions.

No Auto-Tools

Claude can't automatically run scripts, access files, or execute workflows. Everything is manual.

Just You

Instant to Start

Web Search Built-In

Claude can search the web in real-time to answer questions about current events and recent info.

👥

Team Sharing

Slash Commands

03 — ANATOMY

Inside a SKILL.md

Every skill is just a folder with a SKILL.md file. Click each part to learn what it does.

YAML

Frontmatter — the "trigger"

▼
This is how Claude decides when to use your skill. The description field is the most important — Claude reads it and auto-loads the skill when your task matches.
--- name: my-skill description: Use when user asks about deployment or cloud ---
MARKDOWN

Instructions — what Claude does

▼
The body of SKILL.md is plain Markdown. Write step-by-step instructions, rules, examples, or anything you want Claude to follow every time this skill runs.
# Deploy Skill 1. Always run tests first 2. Use docker build 3. Push to registry 4. Run smoke test
SCRIPTS/

Python/Bash scripts

▼
Add executable scripts that Claude runs automatically. Only the output enters Claude's context — not the code itself. This saves tokens and keeps logic clean.
scripts/ validate.py # ← Claude runs this deploy.sh # ← and this
REFERENCES/

Extra documentation

▼
Put large docs, API references, or guidelines here. Claude loads them on demand — only when needed — so they don't waste context window space on every call.
references/ api-guide.md brand-colors.md db-schema.sql
ASSETS/

Templates & files

▼
Store output templates, fonts, images, or boilerplate files here. Claude references them by path — it copies and fills them in rather than generating from scratch.
assets/ report-template.docx logo.png email-template.html
WHERE

Where to put your skill

▼
Personal skills (all projects): ~/.claude/skills/
Project-specific: .claude/skills/ in your repo
Team/shared: commit .claude/skills/ to git
~/.claude/skills/ my-skill/ SKILL.md ← required scripts/ ← optional assets/ ← optional

07 — DEPLOY THIS SITE

Deploy to the Web

Get this interactive guide live at a public URL so your students can access it from anywhere — completely free in under 5 minutes.

1

Create a GitHub Repository

Go to github.com → New repository. Name it ntu-agent-skills. Set it to Public. Don't add README yet.

# In your Codespace terminal: cd ~ git init ntu-agent-skills cd ntu-agent-skills
2

Save this HTML file into the repo

Download the index.html from this guide and place it in your repo folder, or create it directly in your Codespace.

# If you have the file locally: ls index.html # → index.html ✓
3

Push to GitHub

Commit and push your file to GitHub. Replace YOUR_USERNAME with your GitHub username.

git add index.html git commit -m "Add NTU Agent Skills guide" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/ntu-agent-skills.git git push -u origin main
4

Enable GitHub Pages

Go to your repo on GitHub → Settings → Pages → Source: Deploy from branch → main → / (root) → Save. That's it!

# GitHub will show you a green banner: ✓ Your site is live at: https://YOUR_USERNAME.github.io/ntu-agent-skills/ # Takes ~60 seconds to go live
5

Share with students

Copy the URL and share it in your class chat. Students open it in any browser — no login, no install, no cost. Works on phone too!

🎉 Share this link with students: https://YOUR_USERNAME.github.io/ntu-agent-skills/ # Optional: Use Vercel for faster CDN npm i -g vercel vercel --prod ✓ Deployed: https://ntu-agent-skills.vercel.app
⚡

Alternative: One-click Netlify deploy (even easier)

FREE

Go to netlify.com → Add new site → Deploy manually — drag and drop your index.html file. Done. You get a live URL in 10 seconds with zero configuration.
